# Typography

We have Typography components to handle text and styling so that customization can be minimal and usage accurate.

## **Typeface**

**Inter** is a used to establish a clean and readable aesthetic.

<TypefaceCard />

## **Typography Components**

We have four variations of text and each achieves a different purpose.

- `HeadingText`: Used for hero, paragraph and section headings usually followed by long blocks of text.
  Uses `primary` palette as default.
- `BodyText`: The main "content" that users read, like paragraphs, search results, and documentation.
  Uses `secondary` palette as default.
- `MetaText`: It's like a secondary, smaller type of heading (smaller than `BodyText`) used in forms, modals, sidebars, and other aside components. Uses `primary` palette as default.
- `SmallText`: We use this in footnotes, small button text, eyebrow text, tooltips, etc.
  Uses `secondary` palette as default.

### **Overview**

Each Typography component has default text size, styling, font properties, and a default `as`.
Some components also have sizing properties (e.g. `<HeadingText size="LG" />`) and font weight properties (e.g. `<BodyText bold />`).
The default color is based on the intended usage for that text style. For example, headings and meta text are by nature darker and more prominent than body or small text, so we use `gray-800` as the default text color for headings and meta text.
Going further, there are cases where you might need to incorporate an `active` state, a semantic state (like `success` or `error`), or you might need an entirely different color, such as an accent color (very minimal cases will have this).

99.9% of text instances will have a predefined color property, but if you do need to disable all color styles, you can use `disableColorStyles` then set the color class yourself.

## **Text Colors**

The `default` color prop is set for each component with each component using either the `primary` or `secondary` palette as the default text colors.
Aside from `default`, you may choose any palette, or completely override the color styles to use an accent color.

- `primary` (default for HeadingText and MetaText)
- `secondary` (default for BodyText and SmallText)
- `neutral` (used for placeholder text)
- `success`
- `danger`
- `info`
- `tmBlue`
- `white`
- `black`

Text colors are set using the `color` prop which is available to each of the Typography components:

<Canvas>
  <PanelItem align="start" direction="col">
    <MetaText>Default</MetaText>
    <MetaText color="success">Success!</MetaText>
    <MetaText color="neutral">placeholder text</MetaText>
  </PanelItem>
</Canvas>

We also have the option to set a `dark` prop. Each text palette will have a dark version.

<Canvas>
  <PanelItem className="bg-primaryBackgroundDark rounded pl-2" align="start" direction="col">
    <MetaText dark>Primary Dark</MetaText>
    <MetaText color="secondary" dark>
      Secondary Dark
    </MetaText>
    <MetaText color="neutral" dark>
      Neutral Dark
    </MetaText>
  </PanelItem>
</Canvas>

Color styles can be disabled using the `disableColorStyle` prop.
This should be done rarely to never, and will usually be for accent colors. (`EyebrowText` component uses this method.)
Notice that `enableHover` will not operate when color styles are disabled.
Additionally, injecting a `className` into the text component will render the text with a `span` wrapper.

<Canvas>
  <MetaText disableColorStyle enableHover className="text-magenta-500">
    Custom color
  </MetaText>
</Canvas>

#### **Default props:**

- `as: 'h1'`
- `size: 'XL'`
- `bold: false`
- `dark: false`
- `enableHover: false`
- `active: false`
- `disabled: false`
- `color: 'primary'`
